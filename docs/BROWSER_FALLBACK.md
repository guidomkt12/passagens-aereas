# Browser fallback
O serviço `award-scraper` é desativado por padrão. Ele requer login manual por programa e não armazena senha, não contorna CAPTCHA/MFA e não compra, transfere pontos ou emite bilhetes. Perfis persistentes são separados por programa em `SCRAPER_PROFILE_DIR`; `LATAM_SCRAPER_ENABLED`, `SMILES_SCRAPER_ENABLED` e `AZUL_SCRAPER_ENABLED` são independentes. `POST /v1/session/start` abre somente a página de login e retorna a URL para relogin; a busca só ocorre após uma sessão válida. Em erros, screenshot e HTML são gravados em `SCRAPER_ERROR_DIR`; esses artefatos não devem entrar no Git.
